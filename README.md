# نظام أرشفة الوثائق الإلكترونية (Electronic Document Archiving System)

نظام متكامل لإدارة وأرشفة الوثائق الإلكترونية مبني باستخدام PHP مع هيكلية تشبه Laravel.

## المميزات

- ✅ رفع الوثائق الإلكترونية بجميع الأنواع
- ✅ تصنيف الوثائق حسب الفئات
- ✅ البحث المتقدم في الوثائق
- ✅ تحميل الوثائق
- ✅ حذف الوثائق
- ✅ إدارة التصنيفات
- ✅ لوحة تحكم بإحصائيات شاملة
- ✅ واجهة مستخدم عربية جميلة وسهلة الاستخدام
- ✅ قاعدة بيانات SQLite (لا تحتاج إعداد خارجي)

## المتطلبات

- PHP 7.4 أو أحدث
- SQLite (مدمج مع PHP)
- خادم ويب (Apache, Nginx, أو PHP Built-in Server)

## التثبيت والتشغيل

### 1. استنساخ المشروع

```bash
git clone https://github.com/mo3200taha/archiv_php.git
cd archiv_php
```

### 2. إعداد الصلاحيات

```bash
chmod -R 755 storage/
chmod -R 755 database/
```

### 3. تشغيل النظام

باستخدام خادم PHP المدمج:

```bash
php -S localhost:8000
```

أو يمكنك استخدام Apache/Nginx ووضع الملفات في المجلد المناسب.

### 4. الوصول إلى النظام

افتح المتصفح وانتقل إلى:

```
http://localhost:8000
```

## استخدام النظام

### الصفحة الرئيسية
- عرض إحصائيات شاملة (عدد الوثائق، التصنيفات، المساحة المستخدمة)
- عرض آخر الوثائق المضافة

### رفع وثيقة جديدة
1. انقر على "رفع وثيقة" من القائمة
2. أدخل عنوان الوثيقة ووصفها
3. اختر التصنيف المناسب
4. اختر الملف من جهازك
5. انقر على "رفع الوثيقة"

### إدارة التصنيفات
- عرض جميع التصنيفات مع عدد الوثائق في كل تصنيف
- إضافة تصنيفات جديدة

### البحث في الوثائق
- البحث بالكلمات المفتاحية في العنوان والوصف واسم الملف
- تصفية النتائج حسب التصنيف
- تحميل أو حذف الوثائق من نتائج البحث

## قاعدة البيانات

يستخدم النظام قاعدة بيانات SQLite بجدولين رئيسيين:

### جدول categories (التصنيفات)
- id: المعرف الفريد
- name: اسم التصنيف
- description: وصف التصنيف
- created_at: تاريخ الإنشاء

### جدول documents (الوثائق)
- id: المعرف الفريد
- title: عنوان الوثيقة
- description: وصف الوثيقة
- filename: اسم الملف المخزن
- original_filename: الاسم الأصلي للملف
- file_size: حجم الملف بالبايت
- mime_type: نوع الملف
- category_id: معرف التصنيف
- created_at: تاريخ الرفع

## الأمان

- يتم حفظ الملفات بأسماء فريدة لمنع الكتابة فوق الملفات الموجودة
- التحقق من رفع الملفات بشكل صحيح
- استخدام Prepared Statements لمنع SQL Injection
- تصفية المدخلات وإخراج البيانات بشكل آمن
